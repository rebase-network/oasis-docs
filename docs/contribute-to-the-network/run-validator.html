Run a Validator Node

运行验证节点

Oasis Network's Consensus Layer is a decentralised set of validator nodes that maintain a proof-of-stake blockchain.

Oasis Network 的共识层是一组去中心化的验证者节点，它们维护着一个 PoS 区块链。这些节点由独立的节点操作者操作。

因此，它需要一套运行不同功能的分布式节点(包括验证程序节点)。

技术设置

确保你的系统符合硬件的基本条件，Oasis Node 也已经安装。

然后按照运行验证器节点进行操作：

  • 创建 entity。
  • 初始化节点配置。
  • 在你的托管账户中投入足够的资金。
  • 在网络上登记你的entity。

抵押要求

要成为Oasis的验证者，需要在你的托管账户中存入足够的代币。

有关获取entity账的更多信息，请看文档获取账户信息。

目前，你应该已经：

  • 100个ROSE用于entity注册，因为这是当前entity抵押的阈值。

  • 100 ROSE staked for your validator node's registration since that is the current validator node's staking threshold.

  • 100个ROSE用于验证器节点的注册，因为这是当前entity抵押的阈值。

  • 足够的ROSE股份进入前80个实体(以股份形式)，这样验证者将被选举进入共识委员会。

共识委员会（即验证者集）的大小由 max_validators 共识参数配置。

要确定你是否有资格获得 Oasis Protocol 基金会的代理权，请看代理政策文件。
