Handling Network Upgrades

处理网络升级

如果没有网络升级的时候，则按照本指南进行操作会导致你失去当前网络的资格。

当网络同意进行软件升级时，请使用以下指南。

Dump Network State

暂时不要停止Oasis Node 进程。

升级之前，我们将更新升级日志，以指定要转储的区块块高度。

要将网络状态转储到genesis文件中，请运行：

oasis-node genesis dump \
  -a unix:/serverdir/node/internal.sock \
  --genesis.file /serverdir/etc/genesis_dump.json \
  --height <HEIGHT-TO-DUMP>

用我们指定的区块高替换<HEIGHT-TO-DUMP>。

你只能在网络达到 <HEIGHT-TO-DUMP> 区块的高度后才能运行以下命令。

查看当前的区块高度，请运行：

oasis-node control status -a unix:/serverdir/node/internal.sock

并观察consensus.last_height的值。

Patch Dumped State

目前，我们不提供状态补丁。

Download and Verify the Provided Genesis File

从Network Parameters 下载新的 genesis 文件，保存为 /serverdir/etc/genesis.json。

然后将转储状态与下载的genesis文件进行比较：

diff --unified=3 genesis_dump.json genesis.json

我们可以观察到，提供的genesis文件主要更新了某些特定的网络参数。 此外，一些ROSE token已从帐户转移到Common Pool。 其他内容保持不变。 让我们来分析一下变化，并解释一下发生了什么变动。

在网络升级时，以下genesis文件字段总是会改变：

  • chain_id：网络的唯一ID。
  • genesis_time：genesis 文件的有效时间。
  • halt_epoch： 节点停止运行的时间。设置这个值是为了强制升级。

The following fields were a particular change in this upgrade:

下面的字段是此次升级中的特殊改动：

  • staking.params.reward_schedule：描述了权益奖励模型。改为从20%/(年化)开始，在头4年中年化从20%到2%不等。更多细节，请看更新后的token 分发文档。

  • staking.params.disable_transfers: 字段已删除，以实现代币转账。

  • staking.common_pool：代表的是Common Pool。其余额增加了4.5亿ROSE，为增加抵押奖励提供资金。

  • staking.ledger.oasis1qrad7s7nqm4gvyzr8yt2rdk0ref489rn3vn400d6：对应的是 Community and Ecosystem 钱包地址。

  • extra_data：该字段可以保存网络的额外数据，但目前被忽略。此次升级，我们把它改回了Mainnet Beta genesis文件中的值，以包含Oasis的创世引言：“Quis custodiet ipsos custodes?”。[由Oasis社区成员Daniyar Borangaziyev提交]。

Genesis 文件中的余额以nROSE为单位，10亿nROSE相当于1个ROSE代币。更多细节请看Genesis 文件。

如果获得相同的结果，则说明你已成功验证了Genesis文件。

Stop Your Node

这将取决于你的进程管理器。你应该停止Oasis Node进程。

Wipe State

我们不建议你擦除 所有 状态。如果你这样做，可能会丢失节点标识和密钥。

在重新启动节点之前，你应该擦除共识状态。擦除 Node 状态文档中介绍了这个过程。

Update Configuration

如果升级日志提供了更新后的节点配置的说明，请相应更新 /serverdir/etc/config.yml文件。

Upgrade Oasis Node

在再次启动节点之前，请确保将 Oasis Node 二进制程序升级到 Network Parameters 中指定的版本。

Start Your Node

这将取决于你的过程管理器。如果没有流程管理器，你可以选择使用一个。 如果在没有进程管理器的情况下启动节点，可以运行一下命令启动 Oasis Node：

oasis-node --config /serverdir/etc/config.yml

Clean Up

当节点部署成功，你可以清理掉genesis_dump.json文件。
